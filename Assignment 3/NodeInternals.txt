Libuv is cross platform I/O library that handles Asynchronous I/O operations (File System Operations) 
and manages a pool of threads (default: 4 up to 1024).


Question 3:
Node.js handles asynchronous operations through a single-threaded event loop and Libuv,
When you call an async function like fs.readFile(), Node.js registers your callback and delegates the work to Libuv, allowing the main thread to continue without blocking,
Libuv processes the operation,
The event loop monitors for completed operations and executes their callbacks on the main thread when ready. This allows Node.js to efficiently handle thousands of concurrent operations without creating multiple threads, making it ideal for I/O-heavy applications.

Question 4 :
Call Stack: LIFO data structure that tracks function execution,JavaScript executes code from the call stack synchronously
When a function is called, it's pushed onto the stack, when it returns, it's popped off.

Event Queue: A FIFO queue that holds callback functions ready to execute,Stores callbacks from completed async operations, Callbacks wait here until the call stack is empty.

Event Loop: A continuous monitoring mechanism that coordinates between the call stack and event queues
Checks if the call stack is empty, then moves callbacks from the queue to the stack
